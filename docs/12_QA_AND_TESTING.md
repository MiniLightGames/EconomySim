# 12. QA and Testing

## Уровни тестов

### Unit

- доменные правила;
- формулы спроса;
- расчёт кредита;
- логистика;
- price strategy;
- контрактные штрафы.

### Integration

- API + DB;
- создание компании;
- платежи;
- контракт;
- запуск тика;
- миграции.

### Simulation scenario

Сценарии на много тиков:

- дефицит повышает цену;
- логистика создаёт дефицит;
- банкротство банка сжигает вклады;
- война разрушает инфраструктуру;
- реклама влияет на спрос;
- рост денежной массы вызывает инфляцию.

### E2E

- игрок создаёт компанию;
- покупает землю;
- строит facility;
- закупает ресурсы;
- продаёт товар;
- берёт кредит;
- заключает контракт.

## QA gates

Перед merge:

```bash
pnpm lint
pnpm typecheck
pnpm test
pnpm build
```

Перед release:

- e2e green;
- simulation regression green;
- load baseline passed;
- no critical security issues.

## Обязанность QA Agent

QA Agent создаёт недостающие тесты самостоятельно, даже если разработчик их не написал.
