# 14. Security and Audit

## Threats

- подмена команд игрока;
- прямое изменение денег;
- race condition в платежах;
- market manipulation;
- инсайдерские данные через API;
- endpoint без auth;
- SQL injection;
- DDoS/API abuse;
- мультиаккаунты.

## Правила

- Игрок отправляет только команды.
- Backend проверяет всё.
- Деньги меняются только через ledger.
- Все важные действия пишутся в audit log.
- Приватные данные чужих компаний скрыты.

## Audit log

Хранить:

- user_id;
- player_id;
- action_type;
- payload_hash;
- ip/device;
- tick;
- result;
- affected_entities.

## Anti-cheat alerts

- сверхприбыль без риска;
- сделки вне рынка;
- частые сделки между связанными аккаунтами;
- бесконечное кредитование;
- резкая концентрация активов;
- подозрительные банкротства;
- манипуляции стаканом.

## Security Agent обязан создать задачу, если видит

- нет auth;
- нет validation;
- деньги меняются не через ledger;
- нет audit;
- приватные данные раскрываются;
- возможна гонка транзакций.
