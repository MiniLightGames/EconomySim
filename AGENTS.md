# AGENTS.md — главный регламент AI-агентов EconomySim

## Миссия

Довести EconomySim от пустого репозитория до production-ready продукта. Агент не ждёт детальных задач от человека: он сам находит пробелы, создаёт задачи, реализует код, интерфейс, тесты, документацию и проверки.

## Базовый цикл агента

1. **Observe** — изучить репозиторий, docs, тесты, CI, TODO.
2. **Decide** — выбрать следующую самую важную задачу.
3. **Plan** — описать краткий план.
4. **Implement** — написать код, UI, миграции, тесты.
5. **Verify** — запустить проверки.
6. **Report** — зафиксировать результат.
7. **Create Next Tasks** — создать follow-up задачи.

## Обязательные проверки

Если команды отсутствуют — агент обязан создать инфраструктуру для них.

```bash
pnpm lint
pnpm typecheck
pnpm test
pnpm build
```

## Агент не имеет права

- менять деньги без ledger/transaction;
- принимать команду игрока без backend validation;
- скрывать падающие тесты;
- оставлять TODO без задачи в backlog;
- писать UI без error/loading/empty states;
- добавлять pay-to-win;
- ломать публичные API без миграционного плана;
- изменять правила экономики без обновления docs.

## Definition of Done

Задача готова, если:

- код реализован;
- есть тесты;
- UI или API позволяют проверить результат;
- миграции есть, если менялись данные;
- документация обновлена;
- проверки пройдены;
- созданы follow-up задачи.

## Приоритеты

1. Вертикальные срезы продукта.
2. Архитектурная целостность.
3. Корректность экономики.
4. Безопасность денег и активов.
5. UI/UX понятность.
6. Производительность и production hardening.
